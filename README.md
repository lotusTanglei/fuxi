# fuxi (负屃) - 脚本执行与协作管理系统

`fuxi` 是一个专为运维和开发团队设计的脚本管理与执行协作系统。它通过标准化的流程管理，实现了脚本的版本控制、审核、分发执行以及多角色（运维、测试、主管）的线上协作。

## 核心功能

### 1. 脚本生命周期管理 (Script Management)
- **脚本库**: 集中管理所有自动化脚本，支持按标题、状态、类型进行搜索。
- **版本控制**: 脚本的每次编辑都会自动生成新版本，支持查看历史版本内容。
- **在线编辑**: 集成 Markdown 编辑器 (SimpleMDE)，支持代码高亮与文档编写。
- **审核流程**: 开发人员提交脚本 -> 主管(Leader)在线审核（通过/驳回）。

### 2. 执行计划与协作 (Execution Plan)
- **任务包创建**: 将多个通过审核的脚本版本打包成执行计划。
- **多角色分工**:
    - **运维 (OPS)**: 负责在目标现场执行脚本，并提交带有 Markdown 和图片支持的**执行回执**。
    - **测试 (TEST)**: 对执行结果进行验证，确保任务达成预期。
    - **主管 (LEADER)**: 负责计划的最终审计与结项。
- **灵活编辑**: 处于草稿或待执行状态的任务包可由创建者或管理员随时调整。

### 3. 现场与人员配置 (Site & User)
- **现场管理**: 定义不同的目标环境或客户现场。
- **人员绑定**: 支持将用户与特定现场进行关联，实现精细化的权限隔离。
- **用户管理**: 完整的用户增删改查及角色分配（ADMIN, LEADER, OPS, TEST）。

### 4. 个人中心 (User Profile)
- **资料维护**: 用户可自主修改真实姓名等基本信息。
- **安全设置**: 提供原密码校验与新密码修改功能。

### 5. 系统基础 (System Foundation)
- **数据字典**: 灵活配置脚本类型、目标环境等系统级参数。
- **图片上传**: 执行回执支持图片上传，直观记录执行过程。

## 技术栈
- **后端**: Spring Boot 3.2, MyBatis Plus, Spring Security, MySQL
- **前端**: Layui 2.9, Thymeleaf, SimpleMDE
- **工具**: Lombok, Hutool

## 快速启动
1. 确保已安装 Java 17+ 和 Maven。
2. 配置 `src/main/resources/application.yml` 中的数据库连接。
3. 执行启动命令：
   ```bash
   mvn spring-boot:run
   ```
4. 访问地址：`http://localhost:18181` (默认端口)
